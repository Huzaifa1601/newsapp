PULSEWIRE PRESENTATION SHORT NOTES

1. Introduction
PulseWire is a premium full-stack news mobile application built with Flutter, Riverpod, Express, and Firebase. The goal was to create a scalable and visually polished news product instead of a basic content app.

2. Main idea
The app combines a premium reading experience with personalization, backend APIs, and cloud-ready architecture.

3. Frontend
The frontend is built in Flutter with Material 3 and custom theming. I used clean architecture and organized the app into presentation, domain, and data layers.

4. State management
Riverpod manages the app state. This keeps business logic outside the widgets and makes the app easier to maintain and scale.

5. Routing
GoRouter handles the navigation flow from splash to onboarding, authentication, home, search, bookmarks, profile, and article details.

6. Splash and onboarding
The splash screen initializes services and decides the next screen. The onboarding screens introduce the core features before the user enters the app.

7. Authentication
The app supports email and password login, signup, forgot password, Google sign-in, and guest mode. Firebase Authentication is used for secure account handling.

8. Home screen
The home screen is the core experience. It has a breaking news carousel, category tabs, trending news, personalized content, pagination, pull-to-refresh, and skeleton loading.

9. Search
The search feature supports debounced input, suggestions, category filtering, date filtering, popularity sorting, and voice search support.

10. Article detail
The detail screen includes a hero image, metadata, formatted content, share, bookmark, like, related articles, and text-to-speech.

11. Bookmarks and offline support
Bookmarks are cached locally using the local cache service so saved content remains available quickly and supports offline-friendly behavior.

12. Profile
The profile screen includes avatar display, saved count, reading history, preferred categories, theme switching, language setting, and logout.

13. Backend
The backend is built with Node.js and Express. It follows a layered pattern with routes, controllers, services, middleware, and utility files.

14. Backend APIs
There are public APIs for home feed, search, article details, and interactions. There are also admin APIs for content management, categories, users, and analytics.

15. Database
Firestore is used for articles, categories, and interaction records. Firebase Auth manages users. Firebase Storage is prepared for media assets.

16. Security
The backend uses JWT for admin protection, Zod for request validation, Firebase Admin for cloud integration, and structured error handling.

17. Performance
The app uses cached images, local article caching, shared preferences, pagination, and lightweight state updates to improve performance.

18. Scalability
The feature-based structure and clean architecture make the code scalable. Both frontend and backend can grow with additional modules.

19. Future work
Next steps include real push notifications, stronger analytics, more tests, recommendation improvements, and admin dashboard expansion.

20. Closing
PulseWire is not just a UI project. It demonstrates frontend engineering, backend architecture, Firebase integration, state management, performance planning, and product-level thinking.
